# 10 - 语音与音频引擎

## 10.1 音频引擎接口

**文件**: `src/voice/audio-engine-types.ts`

```typescript
interface AudioEngine {
  // 生命周期
  initialize(): Promise<void>;
  destroy(): Promise<void>;
  
  // 麦克风捕获
  startCapture(): void;
  stopCapture(): void;
  isCapturing(): boolean;
  
  // 音频播放
  play(source: AudioPlaybackSource): Promise<void>;
  stop(): void;
  isPlaying(): boolean;
  
  // 音量控制
  toggleMute(): boolean;
  isMuted(): boolean;
  
  // 回调
  setCallbacks(callbacks: AudioEngineCallbacks): void;
}

interface AudioEngineCallbacks {
  onCaptureData: (data: ArrayBuffer) => void;
  onVolumeLevel: (level: number) => void;
  onError: (error: Error) => void;
}

interface AudioPlaybackSource {
  type: string;                      // e.g. "audio/pcm;rate=16000;bits=16"
  size: number;
  arrayBuffer(): Promise<ArrayBuffer>;
}
```

## 10.2 Native 实现

**文件**: `src/voice/audio-engine.native.ts`

**技术**: 使用 `@getpaseo/expo-two-way-audio` 原生模块

**架构**:
- PCM 重采样到 16kHz/16bit（原生模块要求）
- FIFO 播放队列 + 并发控制
- 麦克风数据 → 回调 → VoiceRuntime
- 音量实时监控

**关键实现细节**:
```typescript
const refs = {

};
```

## 10.4 VoiceRuntime 状态机

**文件**: `src/voice/voice-runtime.ts`

**状态定义**:
```typescript
type VoicePhase = 
  | "disabled"    // 不可用
  | "starting"    // 启动中
  | "listening"   // 正在监听
  | "submitting"  // 提交中
  | "waiting"     // 等待响应
  | "playing"     // 播放中
  | "stopping";   // 停止中
```

**主要方法**:
```typescript
class VoiceRuntime {
  startVoice(serverId: string, agentId: string): Promise<void>;
  stopVoice(): Promise<void>;
  handleCapturePcm(data: ArrayBuffer): void;
  handleAudioOutput(source: AudioPlaybackSource): void;
  toggleMute(): boolean;
  
  // 状态快照
  getSnapshot(): VoiceSnapshot;
  subscribe(listener: () => void): () => void;
}

interface VoiceSnapshot {
  phase: VoicePhase;
  isMuted: boolean;
  serverId: string | null;
  agentId: string | null;
  volume: number;
  isSpeaking: boolean;
}
```

**组件**:
- `ContinuousUploader` — 持续上传 PCM 块到服务器
- `PlaybackGroup` — 流式音频块管理 + ack
- `CueSystem` — 思考音效播放

## 10.8 音频数据流

```
麦克风 → AudioEngine.capture
  → onCaptureData callback → VoiceRuntime.handleCapturePcm
    → ContinuousUploader → WebSocket → Server ASR
      → Server response → PlaybackGroup → AudioEngine.play
```
